Variables

Variables can be thought of as named containers.

You can place data into these containers and then refer to the data simply by naming the container.

Variables are used to store/hold values (name = "John") or expressions (sum = x + y).

The data stored could be a text, a number or others.

Declaring (Creating) JavaScript Variables

Creating variables in JavaScript is most often referred to as "declaring" variables.

Before using a variable, you first need to declare it. You have to use the keyword var to declare a variable as follows
var x;

The var statement creates or declare a variable.

Use the var keyword only for declaration or initialization, once for the life of any variable name in a document. You should not re-declare same variable twice.

Assigning a Value to the Variable

Storing a value in a variable is called variable initialization.

You can do variable initialization at the time of variable creation or at a later point in time when you need that variable.

var x;

After the declaration shown above, the variables are empty (they have no values yet)

However, we can declare or create variable first, then assign a value(initialize) later

var x; // declares or creates variable x x = 5; // assigns a value to variable x alert(x) // print the value of x

We can also assign values to the variables when we declare them:

var x = 21; alert(x); // Print the value of x.

the variable x is declared and is initialized to the value of 21.

After the execution of the statement above, the variable x will hold the value 21

Assigning Values to Undeclared JavaScript Variables

If you assign values to variables that have not yet been declared, the variables will automatically be declared.

These statements:

name = "John";

age = 7;

have the same effect as:

var name = "John";

var age = 7;

Declaring Multiple Variables in One Statement

Yes, it is possible to declare multiple variables at once, using a single statement.

Simply separate the variables with a comma (,)

It also enables us to declare multiple variables with the same var keyword as follows:

var name = "John", age = 7;

When you assign a text value to a variable, use quotes (‚Äù) around the value.

Redeclaring JavaScript Variables

If you redeclare a JavaScript variable, it will not lose its original value.

var x = 5;

var x;

console.log(x); // print 5;

After the execution of the statements above, the variable x will still have the value of 5. The value x is not reset (or cleared) when you redeclare it.

The Equal Sign

As you may notice, we used the equal sign to assign values or data to the variables.

The equal sign (=) is called the assignment operator.

The Data Stored/Assigned

The data stored could be a text, a number etc.

These are called data types.

Naming Variables

JavaScript variables should be identified with unique names.

These unique names are called identifiers.

Identifiers can be as short as single letters like a, b, x, y, or z.

Though you can name the variables as you like, it is a good programming practice to give descriptive and meaningful names to the variables.

üí° We should give meaningful names to our variables as much as possible, this helps maintain the readability of our code.

Rules for JavaScript variable names:

- Variable names are case sensitive (y and Y are two different variables)

- Variable names must begin with a letter, an underscore character or a dollar sign.

- A name cannot start with a number.

- Can only contain letters, digits, underscore and dollar signs

- JavaScript reserved keywords ( like var, function, if, else, etc.) cannot be used as variable names

- Variable names **can‚Äôt** include a **mathematical or logical operator** in their name. For instance, 2*something or this+that;

- Variable names**can‚Äôt contain spaces**.

- You‚Äôre not allowed to use any special symbols, like my#num, num%, etc.
