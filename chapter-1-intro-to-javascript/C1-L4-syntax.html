Comments

Comments are used to make programming easier to read and understand. It can be used to add notes, suggestions and hints to JavaScript code

Comments are not executed by the browser

There are 2 types of comments, the single line and the multi-line

Single line comments start with //

// This is a single line comment

Multi-line comments goes between /* and */ 

/*
* This is a 
* Muti-line comment
*/

Statements

A JavaScript statement is a command to a browser. The purpose of the command is to tell the browser what to do. 

document.write("Hello Student");
// this JavaScript statement tells the browser 
// to write "Hello Student" to the web page

The semicolon is optional. the browser is supposed to interpret the end of the line as the end of the statement.
Therefore, semicolon can be omitted if each of your statements are placed on a separate line.

For example, the following code could be written without semicolons 

document.write("Hello World")
document.write("Hello Student")

But when formatted in a single line as follows, we must use semicolon: 

document.write("Hello World"); document.write("Hello Student")

Using semicolons makes it possible to write multiple statements on one line

Although JavaScript does not require to end a statement with a semicolon (;), it is recommended to always use the semicolon to end a statement to make our code more readable and avoid many issues that we may encounter

Following points must be noted regarding JavaScript statements

  • JavaScript statements are executed in the order in which they are written
  • Mutiple line spaces in statement are ignored
  • Statements are separated by semicolon(;) at the end

Case Sensitivity

JavaScript is case sensitive - therefore watch your capitalization closely when you write JavaScript statements

If we mistakenly type Document.Write instead of document.write this statement will throw an error as it did not convey the same meaning to the browser
